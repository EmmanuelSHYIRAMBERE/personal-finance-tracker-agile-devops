# Sprint 0 — Planning

## Product Vision

Build a lightweight Personal Finance Tracker that allows users to manage accounts, record transactions, and monitor financial summaries efficiently.

---

## Product Backlog

### User Story 1: Create Account

As a user, I want to create an account so that I can manage my finances.

**Acceptance Criteria:**

- User can create account with name and initial balance
- Account is stored in system
- Confirmation message is shown

**Priority:** High
**Story Points:** 3

---

### User Story 2: Add Transaction

As a user, I want to add income or expense transactions.

**Acceptance Criteria:**

- User can input amount and type (income/expense)
- Transaction updates account balance
- Transaction is stored

**Priority:** High
**Story Points:** 3

---

### User Story 3: View Balance

As a user, I want to view my account balance.

**Acceptance Criteria:**

- System displays current balance
- Balance reflects all transactions

**Priority:** High
**Story Points:** 2

---

### User Story 4: List Transactions

As a user, I want to see all my transactions.

**Acceptance Criteria:**

- Transactions are displayed in a list
- Each transaction shows amount and type

**Priority:** Medium
**Story Points:** 2

---

### User Story 5: Financial Summary

As a user, I want a summary of income and expenses.

**Acceptance Criteria:**

- System shows total income
- System shows total expenses
- Net balance is displayed

**Priority:** Medium
**Story Points:** 3

---

## Definition of Done (DoD)

A task is considered complete when:

- Feature is implemented
- Code is committed to GitHub
- Unit tests are written and passing
- CI pipeline passes successfully
- Feature meets acceptance criteria
